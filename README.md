**设计模式**

## 单例模式

意图：使程序运行期间只存在一个实例对象
* 简单new struct
* 加锁后new struct
* 判空后再加锁，再new struct
* sync.Once原子操作，once.Do()方法里new struct

## 工厂模式 (factory)

意图: 定义一个创建对象的接口，让其子类自己决定实例化哪一个工厂类，工厂模式使其创建过程延迟到子类进行 主要解决接口选择问题

关键代码: 返回的实例都实现同一接口

应用实例:

您需要一辆汽车，可以直接从工厂里面提货，而不用去管这辆汽车是怎么做出来的，以及这个汽车里面的具体实现。
Hibernate 换数据库只需换方言和驱动就可以。
